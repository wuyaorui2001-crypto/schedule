## 3. AI接手后的标准SOP流程

### 场景A：自然语言添加日程

**用户输入示例：**
- "后天上午10点开会"
- "下个月2号还信用卡"
- "周五下午三点交报告"

**处理流程：**
1. 解析事件信息（标题、时间、优先级、分类）
2. 确认信息（最小化问题）
3. 追加到当前年度文件的 `当月日程明细` 区
4. 运行 `python scripts/recalc.py --update` 更新统计面板

### 场景B：查看日程

**用户输入：**
- "我最近有什么安排？"
- "下周有什么？"
- "查看待办"

**处理流程：**
1. 运行 `python scripts/recalc.py --dry-run YYYY` 查看统计
2. 读取对应年度文件的日程明细
3. 分类展示

### 场景C：完成/取消待办

**处理流程：**
1. 找到对应事件
2. 更新状态为 `[已完成]` 或 `[取消]`
3. 运行 `python scripts/recalc.py --update` 更新统计

## 4. 数据格式

### 日程记录格式
```
- YYYY-MM-DD HH:MM | [状态] | [优先级] | 标题 | #标签
```

### 字段说明
| 字段 | 可选值 | 说明 |
|------|--------|------|
| 状态 | 待办/进行中/已完成/取消 | 任务状态 |
| 优先级 | P0紧急/P1高/P2中/P3低 | 优先级（P0最高） |

### 状态说明
| 状态 | 说明 |
|------|------|
| 待办 | 还未开始的任务 |
| 进行中 | 正在执行的任务 |
| 已完成 | 已完成（可加完成时间） |
| 取消 | 已取消的任务 |

### 示例记录
```markdown
- 2026-03-24 19:00 | [待办] | [P2中] | 取快递 | #生活
- 2026-03-26 15:00 | [待办] | [P1高] | 班会 | #班级 | 提醒:准时
- 2026-03-18 18:00 | [已完成] | [P1高] | 吃饭 | #生活 | 完成:2026-03-18 18:30
```

## 6. 自动化工具

### recalc.py - 统计计算
```bash
# 查看统计（不写入）
python scripts/recalc.py --dry-run 2026

# 更新统计面板（写入文件）
python scripts/recalc.py --update 2026
```

### parser.py - 统一解析
提供标准接口：`parse_record()`, `parse_year_file()`, `get_current_year()`

### config.py - 集中配置
包含状态列表、优先级列表、分类映射等常量。
